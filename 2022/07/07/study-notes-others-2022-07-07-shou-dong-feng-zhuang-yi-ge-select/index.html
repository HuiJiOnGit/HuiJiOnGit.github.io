手动封装一个select

Q: 为什么要手动封装?

  • 用户提出需求是可以根据select中值的不同显示不同的样式, 比如选 true 显示绿色, 选 false 显示红色,一开始我以为能用css自定义的,但是查阅mdn之后发现像select这种高级控件是不支持的

为什么使用 CSS 美化表单组件这么困难?

所以我打算手动封装一个, 思路来源于“小小蚊子”,非常感谢!

由于我这里使用vue, 那正好封装成一个vue组件,直接上代码

Vue.component('c-select', &#123;
    model: &#123;
        prop: 'selectedValue',
        event: 'change'
    &#125;,
    props: &#123;
        placeholder: String,
        selectedValue: Boolean,
        optionType: &#123;
            type: Number | String,
            'default': 1
        &#125;,
        disabled: &#123;
            type: Boolean,
            'default': false
        &#125;
    &#125;,
    data() &#123;
        return &#123;
            isDown: false,
            options: [&#123; key: 1, value: true, label: '通过', class: 'success' &#125;, &#123; key: 2, value: false, label: '失败', class: 'fail' &#125;],
            options1: [&#123; key: 1, value: true, label: '是', class: 'defualt_true' &#125;, &#123; key: 2, value: false, label: '否', class: 'defualt_false' &#125;],
            selectedLabel: '',
            selectedClass: '',
        &#125;;
    &#125;,
    methods: &#123;
        selectMainClick() &#123;
            if (this.disabled) return;
            this.isDown = !this.isDown;
        &#125;,
        itemClick(item) &#123;
            this.isDown = !this.isDown;
            this.selectedLabel = item.label;
            this.selectedClass = item.class;
            this.$emit('change', item.value);
        &#125;,
        selectValueChange(value) &#123;
            var item = this.dynamicOptions.filter(f => f.value === value);
            console.debug(`[CustomSelect Event] selectedValue is $&#123;value&#125;`);
            this.selectedLabel = item[0].label;
            this.selectedClass = item[0].class;
        &#125;
    &#125;,
    computed: &#123;
        dynamicOptions() &#123;
            if (this.optionType.toString() === '1') &#123;
                return this.options;
            &#125;
            return this.options1;
        &#125;,
        disabledClass() &#123;
            return this.disabled ? 'is-disabled' : '';
        &#125;
    &#125;,
    watch: &#123;
        selectedValue(newValue) &#123;
            this.selectValueChange(newValue);
        &#125;
    &#125;,
    mounted() &#123;
        this.selectValueChange(this.selectedValue);
    &#125;,
    template: `
<div style="position: relative;">
    <div class="c-select" @click="selectMainClick">
        <div :class="['c-select__input', selectedClass, disabledClass]">
            <span>
                &#123;&#123;selectedLabel || placeholder&#125;&#125;
                <span class="c-select__input__suffix">
                    <i class="fa fa-angle-down" aria-hidden="true"></i>
                </span>
            </span>
        </div>
    </div>
    <ul class="c-select-dropdown__list" :style="&#123;'display': isDown ? 'block': 'none'&#125;">
        <li :class="['c-select-dropdown__item', item.class]" v-for="item in dynamicOptions" :key="item.key" @click.prevent.stop="itemClick(item)">&#123;&#123;item.label&#125;&#125;</li>
    </ul>
</div>
`
&#125;);

使用

<c-select v-model=""></c-select>

实现的关键点

  1. 使用了v-model绑定, 在组件里面使用model选项修改默认的v-model行为.
  2. 当子元素是绝对定位的时候,宽度是会跟相对定位的父元素保持一致,如果父元素没有相对定位的,那就是body的宽度了,所以你能看到我在根节点加了相对定位

不过这个东西还不算成熟,还有很多缺陷

  • 客户的需求没有更复杂,所以就把数据源都写里面了,没有用prop,更好应该使用prop的
  • 就是这里面有个问题就是点击了弹出下拉选择项之后,你点其他位置,非组件元素,下拉选择是不能主动消失的,和原生组件的行为不一样,这个暂时还没有好想法去实现
  • 就是下拉生成的位置其实不要在那里,放到body下面可能会更好,不懂为什么,看人家ele是这样干,我想应该有他的道理

2022-08-26更新,问了下朋友之后采用监听全局的点击来解决第二个问题点

<script setup>
import &#123; ref, onMounted &#125; from "vue";

const num = ref(0);
const msg = ref("Hello World!");
let show = ref(false);
const popper = ref(null);
const btnClick = (e) => &#123;
  //点击下拉列表的选项之后就隐藏下拉
  show.value = !show.value;
&#125;;
const globalClick = (e) => &#123;
  msg.value = "global know you click";
  num.value++;
  //判断点击事件的触发是否在下拉列表中,如果不是,那就隐藏下拉列表
  if (!e.path.some((s) => s === popper.value)) &#123;
    show.value = false;
  &#125;
&#125;;
onMounted(() => &#123;
  //监听全局的点击事件,注意是捕获阶段,第三个参数根据mdn的推荐是用一个对象来传参,虽然很多旧文章都是直接使用一个true,但是按mdn的来使用比较好
  document.addEventListener("click", globalClick, &#123;
    capture: false,
  &#125;);
&#125;);
</script>

<template>
  <h1>&#123;&#123; msg &#125;&#125;</h1>
  <h4>
    &#123;&#123; num &#125;&#125;
  </h4>
  <button @click.stop.prevent="btnClick">click me!</button>
  <div ref="popper" class="popper" v-show="show">look at me!</div>
</template>
<style scope>
.popper &#123;
  width: 100px;
  height: 50px;

  border: 1px solid red;
&#125;
