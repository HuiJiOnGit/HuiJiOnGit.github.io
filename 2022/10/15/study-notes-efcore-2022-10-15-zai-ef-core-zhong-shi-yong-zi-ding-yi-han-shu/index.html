在ef core中使用自定义函数

在有些时候,我们因各种原因不得不使用数据库函数的时候, 真希望ef直接全部都支持, 但实际这是太过理想了, 但是ef本身也留下了自定义函数的路子, 下面我来记录下我的尝试过程

1. 首先肯定是搜索啊

于是乎我从官网和博客园找到想要的内容

EFCore3.1+编写自定义的EF.Functions扩展方法

用户定义的函数映射

下面简单介绍下官网的, 3 步就ok了

  • 在数据库创建自定义函数(或者使用数据库已存在的函数)
CREATE FUNCTION dbo.CommentedPostCountForBlog(@id int)
RETURNS int
AS
BEGIN
    RETURN (SELECT COUNT(*)
        FROM [Posts] AS [p]
        WHERE ([p].[BlogId] = @id) AND ((
            SELECT COUNT(*)
            FROM [Comments] AS [c]
            WHERE [p].[PostId] = [c].[PostId]) > 0));
END
  • 为了在 EF Core 中使用此函数，我们将定义以下 CLR 方法，并将其映射到用户定义的函数(注意这里不需要实现, 在.net中只是使用这个方法的定义, 不会真的调用这个方法, 建议抛出InvalidOperationException或NotSupportedException)：
public int ActivePostCountForBlog(int blogId)
    => throw new NotSupportedException();

这里不得不提下EF.Funtions这个扩展定义的方法,我本来也是想将方法定义到这个下面的, 想和ef保持统一的, 但是后面发现有点问题,具体可以看看这个issue

  • 此函数定义现在可以与模型配置中用户定义的函数关联：
modelBuilder.HasDbFunction(typeof(BloggingContext).GetMethod(nameof(ActivePostCountForBlog), new[] { typeof(int) }))
    .HasName("CommentedPostCountForBlog");

总之就是不要定义成扩展的

执行查询和生成的语句如下:

var query1 = from b in context.Blogs
             where context.ActivePostCountForBlog(b.BlogId) > 1
             select b;
/*
SELECT [b].[BlogId], [b].[Rating], [b].[Url]
FROM [Blogs] AS [b]
WHERE [dbo].[CommentedPostCountForBlog]([b].[BlogId]) > 1
*/

可以看到在语句中就直接使用了这个函数, 除了在where中使用,也可以在 select 中使用, 可见是十分灵活的

记录下我需要的函数映射

由于数据库存在很多多对多关系是在一个字段中保存的a;aa;aaba,像这样的,这样查询起来非常麻烦,非常难写语句, 后面了解到pg其实是有一种Array类型的,这个类型就可以非常方便的查询包含关系, 像上面的例子, string.Contains('aa')查出来就是错的,但是转为sql的话,使用'aa' = ANY(string_to_array(string))的话,就很方便查了
但是ef并不直接支持string_to_array和ANY的查询, 是string类型不支持这样查, 如果映射到.net是array类型的时候是可以支持的,至于为什么没有正确映射为数组,别问为什么,呵呵(肯定不是建表那个古板又落后).

public static class DbFunctionsExtensions
{
    public static string[] StringToArray(string source, string separator)
    => throw new NotSupportedException(nameof(StringToArray));

    public static bool Any(string element, string[] source)
    => throw new NotSupportedException();
}

//在dbcontext的OnModelCreating方法中定义如下内容
modelBuilder.HasDbFunction(typeof(DbFunctionsExtensions).GetMethod("StringToArray", new Type[] { typeof(string), typeof(string) }), b =>
{
    b.HasName("string_to_array");
});

modelBuilder.HasDbFunction(typeof(DbFunctionsExtensions).GetMethod("Any", new Type[] { typeof(string), typeof(string[]) }), b =>
{
    //b.HasName("ANY"); 这里就不用定义name了,简单的定义name只能做到简单的映射, 像这个any的语法真的很奇葩, 所以要自己翻译这个方法,不能用默认的
    b.HasTranslation(args =>
        new PostgresAnyExpression(
            args[0],
            args[1],
            PostgresAnyOperatorType.Equal,
            args[0].TypeMapping
        )
    );
});

'aa' = ANY(array[])
这里的翻译真是一波三折,难点全在这里了,一开始我也是使用SqlBinaryExpression来生成表达式树的,但是中间那个=不能用Equal翻译的,会翻译成==,
如果用Assign, 则是SqlBinaryExpression不支持这个操作,后面实在没辙,去观看了efcore.pg的源码,终于功夫不负有心人,
找到了官方定义的PostgresAnyExpression的表达式,使用这个就可以正确翻译这个any的语法

翻译后的语句如下

SELECT ...
FROM s_user AS s
WHERE 'FS' = ANY (string_to_array(s.site_id, ';'))
